# ぱくぱくタイマー 要件メモ

## アプリの目的

子どもが楽しみながらご飯を食べられる、iPad横向き用の食事タイマーアプリ。
タイマーが進むたびに中央のキャラクター「パクパクくん」の口がパクパク動き、食べ進めている感じが視覚的にわかる。

## 共通仕様

### 対象と画面

- iPad横向きで使う。
- PWA版の見た目基準は、repo内の現行 `index.html`、`styles.css`、`app.js` とする。
- SwiftUI版の見た目基準は、実装開始時点の現行PWA画面に準拠する。フォルダ内の参照画像 `assets/references/pakupaku-timer-design-reference.png` は初期案の参考にとどめ、SwiftUI版では参照画像より現PWAの配置・寸法・状態表示を優先する。
- 画面上部にタイトル「ぱくぱくタイマー」を表示する。
- タイトルの下に、全体の残り時間を表示する。
- 中央に大きな丸いタイマーを表示する。
- 大きな丸は3つの食べ物エリアに分ける。
- 画面全体の背景に、小さな食べ物柄を均等に表示する。

### 食べ物エリアの配置

- 左: ヨーグルト
- 真ん中: 卵焼き
- 右: ご飯

### 食べる順番と初期時間

- 食べる順番: ご飯 -> 卵焼き -> ヨーグルト
- 初期時間:
  - ご飯: 5分
  - 卵焼き: 5分
  - ヨーグルト: 4分
- 全体時間は、3つの時間の合計をもとにした残り時間として表示する。

### タイマーの動き

- 各エリアは、その食べ物に割り当てた時間に合わせてだんだん減っていく。
- ご飯、卵焼き、ヨーグルトの順に食べ進める。
- ご飯、卵焼き、ヨーグルトが終わるたびに自動では次へ進まず、区切り確認待ちでタイマーを止める。
- ご飯、卵焼きの区切り確認待ちでは「ごはん おしまい？」「つぎは 卵焼き」のように、終わった食べ物と次の食べ物を表示する。
- 区切り確認待ちで「つぎへ」を押すと、次の食べ物からタイマーを再開する。
- 区切り確認待ちで「もう1分」を押すと、終わった食べ物の現在のタイマー時間だけを1分延長して、同じ食べ物を続ける。
- 「もう1分」で延長している間だけ「完了」ボタンを表示する。
- 「もう1分」の延長分は設定値には保存しない。
- ヨーグルトが終わったときは自動で完了表示へ進まず、区切り確認待ちでタイマーを止める。
- ヨーグルトの区切り確認待ちでは説明文や見出しを表示せず、「ごちそうさま」と「もう1分」だけを表示する。
- ヨーグルトの区切り確認待ちで「ごちそうさま」を押すと、完了表示へ進む。
- ヨーグルトの区切り確認待ちで「もう1分」を押すと、ヨーグルトだけを1分延長して再開する。
- ご飯・卵焼き延長中に「完了」を押した場合は、延長中の食べ物を途中終了して次の食べ物から再開する。
- ヨーグルト延長中に「完了」を押した場合は、次の食べ物へ進まず完了表示へ進む。
- 大きな遅延やバックグラウンド復帰で複数区間をまたいだ場合も、一度に飛ばさず最初に到達した区切りで止める。

### かかったじかん

- すべて終わったら、食事タイマー完了としてわかりやすく表示する。
- 完了後は「かかったじかん」として、ご飯、卵焼き、ヨーグルトそれぞれの実際にタイマーが動いていた時間をまとめて表示する。
- 「かかったじかん」には、一時停止中、区切り確認待ち中、設定画面を開いている時間を含めない。
- 「もう1分」で延長した場合も、実際にタイマーが動いた秒数だけを対象の食べ物の「かかったじかん」に含める。
- 延長中に「完了」を押した場合、残っていた延長分は「かかったじかん」に含めない。
- 実績時間は1分未満なら「30秒」、分単位なら「6分」、端数がある場合は「5分20秒」のように表示する。

### パクパクくん

- 大きな丸の中心に「パクパクくん」を表示する。
- タイマーが動いている間、口を開いたり閉じたりしてパクパクする。
- 区切り確認待ち中は通常のパクパク動作を止める。
- 完了時は通常のパクパク状態を止め、正面向きのニコニコ顔で食べ終わったことが伝わる表情にする。

### 音

- ご飯、卵焼き、ヨーグルトがそれぞれ終わるたびに、短い区切り音を鳴らす。
- 完了時にも短い区切り音を鳴らす。
- 音は短く、楽しい印象にする。
- 子どもが驚きすぎない音量・音色にする。
- 音は画面内の音ボタンでオン・オフできる。

### 設定

- それぞれの時間を1〜60分で変更できるようにする。
- 設定できる項目:
  - ご飯の時間
  - 卵焼きの時間
  - ヨーグルトの時間
- 設定画面を開いたときは、現在タイマーに適用中の時間を表示する。
- 設定画面を開いている間はタイマーを止める。
- 設定確定時に設定値が変わっていた場合だけ、全体の時間表示と各食べ物の残り時間を更新する。
- 設定値が変わった場合だけ、テストモードは解除される。
- 設定値が変わった場合は、区切り確認待ちとその場限りの「もう1分」延長を解除する。

### 操作ボタン

- スタート: タイマーを開始する。
- とめる: タイマーを一時停止する。
- もう一回: 完了後に最初から開始する。
- リセット: タイマーを初期状態に戻す。
- テスト: 各食べ物を5秒ずつにして動作確認できる。
- つぎへ: 次の食べ物から再開する。
- ごちそうさま: ヨーグルト確認パネルを閉じて完了表示へ進む。
- もう1分: 終わった食べ物を現在のタイマー状態だけ1分延長して再開する。
- 完了: もう1分で延長している間だけ表示する。

## PWA版のみ

- 既存PWAの `index.html`、`styles.css`、`app.js`、`manifest.webmanifest`、`service-worker.js` は削除・置換しない。
- iPadのホーム画面に追加したとき、アプリ名とテーマカラーが反映される。
- GitHub PagesなどHTTPSで公開した場合、初回読み込み後は主要ファイルをキャッシュし、Macを閉じていてもiPadから再表示できる。
- iPad Safariで音が鳴るように、スタートや音ボタンのタップ時に音声再生の準備をする。
- オフライン再表示用に、HTML、CSS、JavaScript、マニフェスト、アイコンをService Workerで保存する。
- 同じ画面内の設定パネルで、それぞれの時間を1〜60分のプルダウンから変更できるようにする。
- 設定ボタンの再タップ、または設定パネル外側のタップでパネルを閉じたとき、選択中の時間をタイマーへ反映する。

## SwiftUI版のみ

- repo直下に `PakuPakuTimer.xcodeproj` とSwiftUIアプリ本体を追加する。
- SwiftUI版はWebView包装ではなく、現行PWAの共通仕様をネイティブで再現する。
- SwiftUI版は、PWAの4:3 `.stage` を基準サイズ `1366 x 1024` の固定座標キャンバスとして再構成する。
- SwiftUI版は、利用可能領域内で `scale = min(width / 1366, height / 1024)` を使って固定キャンバス全体を拡大縮小し、safe area内の中央に配置する。
- 固定キャンバス外の余白はPWAと同じ緑系背景で塗り、黒帯を出さない。
- PWA CSSの主要値であるステージ枠、角丸、背景グラデーション、草、旗、中央プレート、3分割セグメント、点線アウトライン、食べ終わり色、左右ボタン、右側パネル、下部操作ボタン、確認・設定・完了パネルはSwiftUI側の定数・Shape・Viewへ移植する。
- 動的要素であるタイトル、全体時間、各食材ラベル、残り時間、セグメント進行、確認待ち、設定、完了表示、パクパクくん本体と口アニメーションはSwiftUIで描画する。
- `Assets.xcassets` を作成し、Xcode projectのResourcesへ登録する。画像化する場合は、静的な食べ物背景柄と固定装飾だけに限定し、タイマー状態に応じて変わる表示や操作UIは画像化しない。
- パクパクくんはPWAの横向き顔構造に合わせてSwiftUI Shapeで作る。顔の円内で口を必ずclipし、通常の開き口、閉じ口、完了時のニコニコ状態を再現する。
- Bundle IDは `dev.vanntoru.PakuPakuTimer` とする。
- 対象はiPadのみとし、ビルド設定で `TARGETED_DEVICE_FAMILY = 2` を使う。
- 対応向きはInfo.plistでiPad横向きに明示する。
- iPad横向き専用表示として扱うため、Info.plistでフルスクリーン必須を明示する。
- 中心構成は `FoodKind`、`MealTimerViewModel`、`MealTimerView`、`SoundPlayer` に分ける。
- `FoodKind` は、ごはん、卵焼き、ヨーグルトの順番、表示名、初期時間、表示色を持つ。
- `MealTimerViewModel` は、開始、一時停止、リセット、テスト、つぎへ、もう1分、ごちそうさま、延長中の完了、設定変更、音オン/オフを管理する。
- 時間進行はテストしやすいように、ViewModelへ `advance(by:)` の進行口を用意する。
- 実時間のTimerはSwiftUI View側から `advance(by:)` へ差分秒数を渡す。
- `SoundPlayer` はアプリ内で短い区切り音を生成して再生する。
- 最初のゴールは、自分のiPad実機にXcodeから入れて使える状態にすること。
- App Store提出用スクリーンショット、Privacy Nutrition Label、審査文言は次段階に回す。

### SwiftUI版の実機Run手順

1. Xcodeで `PakuPakuTimer.xcodeproj` を開く。
2. Schemeに `PakuPakuTimer` を選ぶ。
3. Run先に自分のiPad実機を選ぶ。
4. Target `PakuPakuTimer` の Signing & Capabilities で自分のTeamを選ぶ。
5. Runして、横向き画面、音、設定、確認待ち、完了表示を手動確認する。

## テスト方針

- XCTestでViewModelを検証する。
- ごはん終了で卵焼きへ自動進行せず、確認待ちになること。
- `つぎへ` で次の食べ物から再開すること。
- `もう1分` は設定値を変えず、その食べ物だけ延長すること。
- 延長中だけ `完了` が表示対象状態になること。
- ごはん・卵焼き延長中の `完了` は次の食べ物へ進み、ヨーグルト延長中の `完了` は完了表示へ進むこと。
- ヨーグルト終了は完了表示へ自動遷移せず、`ごちそうさま` 待ちになること。
- 一時停止中、確認待ち、設定中の時間が `かかったじかん` に含まれないこと。
- 14分以上など大きな時間ジャンプを与えても、ごはん終了確認で止まり、卵焼きやヨーグルトへ飛ばないこと。
- 設定変更時にテストモード、確認待ち、延長状態が解除されること。
- `xcodebuild test` を `iPad Pro 13-inch (M5)` Simulator で実行する。
- `xcodebuild build` でSimulator向けビルド成功を確認する。
- 既存PWA側は `git diff --check` を実行し、必要ならローカル表示で壊れていないことを確認する。
